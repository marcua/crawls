Not sure on size (merm would be able to tell you), but I'm guessing that it can't be a large picture (i.e. 1 MB - that's huge - generally they should be <100 KB, but again, merm would know for sure).

The image itself will resize automatically so the dimensions, I've been told, don't matter greatly. I personally like to make the image "just so" so it won't be pixilated - something like 100x100px or 80x80px.

As far as image type - that's pretty easy for me to answer. It's the web so it should be a GIF or JPEG (a.k.a. JPG). BMP (bitmap) is still used, but from a web designer's standpoint, not widely used anymore - which means it may or may not be accepted here (merm would know for sure). PNGs are starting to be used more widely as well, but the transparency is not always recognized by all browsers. By default go with GIF or JPEG.

I will point out here that sometimes people have image types on their computers that they often try to share with others but that are unique to their camera software (which means anyone without that camera software won't be able to view it). If this is the case you'll need to use that camera software to convert it to the above said GIF or JPEG - I'm pretty sure the forum won't allow you to upload a file with an unrecognizable extension since no one will be able to view it.

Hope that helps.
