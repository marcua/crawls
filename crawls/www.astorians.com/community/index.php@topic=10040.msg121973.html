I'll fill in for a message, since murph seems to be elsewhere.

The Download size: 0 indicates that there are no additional things for Windows Update to download before it begins the installation. Windows Update has an option to download updates automatically, then present them for you to install.  Or perhaps these updates were downloaded when you tried to install them before, and it had determined that it won't download them again.

I haven't tried to install either of these yet - I don't use my Windows machine at home for much but the cat-cam and doing taxes. The fact that both updates fail hints that the problem is not with the updates themselves.

Windows Explorer crashes are a pain in the neck. Window Explorer is the thing that manages directory, er excuse me, folder windows, and also manages all the icons on the "desktop", the desktop toolbars, and the "Start" button. That's why they all disappear when it crashes. It is so central to Windows operation that Windows starts a new copy if it fails, which is why everything reappears a little while later.

Third-party programs often use a feature that lets them hook into Explorer, to present their features as if they were part of the file system. These include cd/dvd burners, digital cameras, video cams, music management software programs like iTunes, etc.  Some virus and spy-ware programs use this feature too.  You may have a broken or failed installation of one of these things that is crashing Explorer.

If it were mine, I would check the event-viewer for clues of things that are logging messages as they fail.  Then I would "uninstall" (via the control-panel "add and remove" feature) any software installed just before the explorer problem appeared, and also run an anti-virus program "full-scan" of the machine. Beyond that, finding the root of such problems involves (1) checking web sites for the stuff that's installed to find admissions of guilt, and (2) finding a nerdy kid to do technical detective work, and (3) having good luck.
